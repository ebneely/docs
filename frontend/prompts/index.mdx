# Prompts

Frontend prompts run with Claude Design (or whichever visual-aware tool you use). They focus on design tokens, component generation, polish review, and accessibility audit. The shape mirrors the backend prompt pages — when to use, the prompt itself, what comes back, what to do next — but the artifacts are mockups and components, not specs and code.

<Note>
Most concrete frontend prompts live inside the design-token variant pages. Each variant has setup, component-generation, and polish-review prompts tuned to its aesthetic. Start there if you need a copy-paste-ready prompt.
</Note>

## Where these fit in context

Frontend prompts assume you've already picked a token variant. If you haven't, [start there](/frontend/design-tokens/index). The token choice constrains every prompt downstream — a component-generation prompt without locked tokens produces generic output.

For the philosophical anchor, see [Design first](/frontend/mindset/design-first).
