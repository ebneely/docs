</Note>

## Overview

Anchored on the visual language of [Vercel](https://vercel.com) — the dashboard, marketing site, and dev tools. The aesthetic is minimalist, sharp, and high-contrast, with dark as the default mode.

Use this variant when the product is technical, the audience is developer-shaped, and the surface should feel precise rather than friendly. The variant rewards dense, geometric UI with crafted defaults: pure-or-near-pure black anchors, borders carrying elevation, snappy motion, and a single restrained accent.

## The aesthetic in detail

- Dark-default — Vercel ships dark first because the audience lives in terminals. Pure-or-near-pure black backgrounds anchor every other color against an extreme.
- Backgrounds and surfaces sit at the bottom of the lightness scale; borders are mid-gray; text is near-white. The contrast budget is spent on type, not on chrome.
- Borders carry elevation, not shadows. Resting surfaces sit on the same plane separated by a 1px border. Drop shadows reserved for genuinely floating UI: popovers, modals.
- Single restrained blue accent, used only on primary CTAs and focus rings. Never decorative.
- Geist sans with Geist Mono pairing. The mono family is non-decorative — it renders code, terminal output, and ID-shaped values to signal "this is literal".
- Tight radius scale (4-8px max). Larger radii soften the surface and break the geometric tone.
- Snappy motion in the 100-150ms range with sharp ease-out curves. Slower than that reads sluggish on a high-density technical surface.
- Restraint in color use — a single accent at one saturation, no gradient hero treatments, no rainbow status arrays.

## Token categories covered

Color (dark default plus light override, single accent, status colors), typography (sans + mono families, size scale, weight, leading, tracking), spacing (4px-base scale), sizing (control heights, container widths), radius (tight scale capped at 8px), shadows (encoded as 1px borders for elevation, real drop shadows reserved for modal layers), motion (durations, easing).

Vercel emphasizes border-encoded elevation and mono pairing — the variant should preserve both.

## Claude Design prompts

### Setup prompt

```
You are setting up a frontend project anchored on Vercel's aesthetic:

- Dark-default. Pure-or-near-pure black backgrounds, mid-gray borders,
  near-white text. Contrast budget spent on type, not chrome.
- Single restrained blue accent, used on primary CTAs and focus rings only.
  Never decorative.
- Geist sans with Geist Mono pairing. Mono renders code, terminal output,
  and ID-shaped values.
- Borders carry elevation, not shadows. Resting surfaces use 1px borders.
  Real drop shadows only for modals and popovers.
- Tight radius scale (4-8px max). No larger radii.
- Snappy motion (100-150ms range) with sharp ease-out curves for almost
  everything. Slower only for modal transitions.
- Restraint in color: single accent at one saturation, no gradients, no
  rainbow status arrays.

Generate a complete tokens.json covering color (dark default plus
[data-theme="light"] override, single accent, status colors), typography
(sans + mono families, size, weight, leading, tracking), spacing, sizing,
radius (capped at 8px), shadows (encoded as 1px borders for elevation),
motion, and easing. Values should reflect the aesthetic above.

Then output the same tokens as CSS custom properties on :root (dark by
default, not wrapped in a media query) with a [data-theme="light"]
override flipping bg/surface/border/text scales. Set color-scheme: dark
on :root.

Add a base typography reset and a minimal CSS reset. No component
styles. Output the full file contents.
```

### Component prompt

```
You are generating a [COMPONENT_TYPE] anchored on Vercel's aesthetic:

- Minimalist, sharp, high-contrast. Dark-default.
- Borders carry elevation, not shadows. Single restrained blue accent
  on primary CTA and focus only.
- Geist sans with Geist Mono on code, IDs, and shortcut keys.
- Tight radius (4-8px). Snappy motion (100-150ms) with ease-out.

Use only token references (assume tokens like --color-bg, --color-surface,
--color-border, --color-border-strong, --color-text, --color-text-muted,
--color-accent, --space-*, --radius-*, --shadow-*, --duration-*, --easing-*,
--font-sans, --font-mono exist). No raw hex codes. No raw px values
except where a token does not exist.

Cover all states: default, hover, focus-visible, active, disabled, loading,
error (where applicable), empty (where applicable). Resting surfaces use
1px solid var(--color-border). Reserve box-shadow for floating UI only.
Focus-visible uses --shadow-focus (a 2px ring of var(--color-accent)).
Render any code, ID, or shortcut key in var(--font-mono). Transitions use
--duration-fast or --duration-medium with --easing-out.

Output as HTML+CSS or React+CSS, single file.
```

### Review prompt

```
You are reviewing a UI component against Vercel's aesthetic standards:

- Border-not-shadow elevation: resting surfaces separated by 1px borders,
  not shadows. Flag box-shadow on non-floating UI.
- Single-accent discipline: blue accent on primary CTA and focus only.
  Flag decorative accent use or multi-hue palettes.
- Motion timing: 100-150ms standard, ease-out by default. Flag anything
  outside 75-220ms without reason. Flag linear or default easing.
- Radius restraint: 4-8px max. Flag radii above 8px on technical surfaces.
- Mono pairing: any code, ID, terminal content, or shortcut key rendered
  in --font-mono. Flag sans-only on literal values.
- Spacing rhythm: all values pull from the 4px-base scale. Flag arbitrary
  numbers (7px, 13px, 22px).
- State coverage: every interactive element defines default, hover,
  focus-visible, active, disabled. Loading and error where applicable.
- Color restraint: single accent at one saturation. Flag gradient hero
  treatments or rainbow status arrays.

List violations as: Block / Fix-First / Note. Each violation cites the
dimension and the suggested fix.
```
