## Overview

Anchored on the visual language of [Tailwind CSS](https://tailwindcss.com)'s default theme. The aesthetic is generic-but-opinionated — well-documented defaults that compose cleanly with the wider Tailwind ecosystem.

Use this variant when you want maximum flexibility, when the team already runs Tailwind, or when the project's visual identity has not yet been chosen. The variant gives competent neutral output with no strong stylistic posture — that is the trade-off. Pick a different variant if the product needs character (premium-warm like Stripe, sharp-technical like Vercel).

## The aesthetic in detail

- Light-default with class-strategy dark mode (`<html class="dark">`). Lets the user override OS preference, which is more correct than `prefers-color-scheme` for tools.
- One neutral scale chosen at setup and held across the project: slate (cool), gray (true-neutral), zinc (slight warmth), neutral (chromaless), or stone (warm). Mixing reads as accidental.
- 11-step neutral ramps (50 / 100 / 200 / ... / 900 / 950) plus accent ramps for blue, red, green, amber. Generous step count gives room for nuanced contrast without picking custom values.
- Inter sans with JetBrains Mono pairing — neutral, well-tested, broadly available. Type sizes follow a consistent scale with paired leading values.
- Generous, well-spaced radius scale (sm / DEFAULT / md / lg / xl / 2xl / 3xl / full). Radius can communicate friendliness; the project picks a ceiling and stays under it.
- Layered drop shadows (sm / DEFAULT / md / lg / xl / 2xl / inner) — Tailwind uses real shadows for elevation, not borders. This is the major break from Vercel/Linear.
- Eight motion durations (75 / 100 / 150 / 200 / 300 / 500 / 700 / 1000 ms) and four easings (linear / in / out / in-out). Avoid `linear` — it is a rookie tell.
- Opinion-light by design. The variant ships strong defaults but no editorial posture. Add character via accent choice and radius ceiling.

## Token categories covered

Color (11-step neutral ramp plus blue/red/green/amber/white/black accents), typography (sans + mono families, size scale with paired leading, weight), spacing (extensive 4px-base scale from `0` through `96`), sizing (control heights), radius (full Tailwind scale), shadows (full layered scale plus `inner` and `none`), motion (eight duration steps), easing (linear + in + out + in-out).

Tailwind emphasizes interop — these tokens should map cleanly to `tailwind.config.js` so utilities like `bg-slate-900`, `text-blue-600`, `p-6` resolve without translation.

## Claude Design prompts

### Setup prompt

```
You are setting up a frontend project anchored on Tailwind CSS's default theme:

- Light-default. Dark mode via the class strategy (<html class="dark">),
  not prefers-color-scheme.
- One neutral scale, picked at setup: slate (cool) / gray (true-neutral) /
  zinc (slight warmth) / neutral (chromaless) / stone (warm). Hold it
  across the project — never mix.
- 11-step ramps for the chosen neutral plus blue, red, green, amber.
- Inter sans with JetBrains Mono pairing. Size scale with paired leading.
- Generous radius scale (sm / DEFAULT / md / lg / xl / 2xl / 3xl / full).
  Pick a project-wide ceiling and stay under it.
- Layered drop shadows for elevation — this is the Tailwind way.
- Eight duration steps (75-1000ms) and four easings. Never linear.

Generate the tokens in two coordinated places:

1. As CSS custom properties on :root in globals.css (for non-utility
   consumers).
2. As theme.extend in tailwind.config.js so utilities like bg-slate-900,
   text-blue-600, p-6 resolve to the same values without translation.

Set darkMode: 'class' in the config and provide a .dark override on :root
that flips background, surface, border, and text scales.

Confirm both globals.css and tailwind.config.js are wired before
generating any component.
```

### Component prompt

```
You are generating a [COMPONENT_TYPE] anchored on Tailwind's defaults.

Aesthetic: opinionated Tailwind defaults. Light by default, dark via the
class strategy.

Constraint: Tailwind utility classes only. No inline styles. No custom
CSS classes outside utilities. No arbitrary values like p-[13px] — every
value comes from the scale.

Cover all states: default, hover (hover:), focus-visible (focus-visible:),
active (active:), disabled (disabled:), loading (where applicable). Pair
a dark: variant on every relevant utility (dark:bg-slate-900,
dark:text-slate-100) — no leaks where dark mode reveals light-only colors.

Constraints:
- Spacing from the 4px-base scale (p-1 through p-24). No arbitrary values.
- Radius from {rounded-sm, rounded, rounded-md, rounded-lg, rounded-xl,
  rounded-2xl, rounded-3xl, rounded-full}. Hold the project ceiling.
- Shadow from {shadow-sm, shadow, shadow-md, shadow-lg, shadow-xl,
  shadow-2xl, shadow-inner}. Tailwind uses real shadows for elevation.
- Motion via {transition, duration-{75|100|150|200|300|500|700|1000},
  ease-out|ease-in|ease-in-out}. Never ease-linear.
- One neutral scale (chosen at setup) plus blue/red/green/amber for
  accents and status. Do not mix neutrals.
- Type sizes from {text-xs, text-sm, text-base, text-lg, text-xl,
  text-2xl, ...}. No arbitrary sizes.

Return as plain HTML or a React component. Markup only — no extracted CSS.
```

### Review prompt

```
You are reviewing a UI component against Tailwind's polish bar:

- Utility-class consistency: every spacing, color, radius, shadow, and
  motion value comes from a Tailwind utility. Flag arbitrary values
  like p-[13px], bg-[#abc], duration-[230ms].
- Spacing scale: all gaps and padding from the 4px-base scale (p-1
  through p-24). Flag one-offs.
- Radius ceiling: rounded values from the chosen ceiling and below
  (project picks one of sm / md / lg / xl / 2xl / 3xl). Flag values
  above the ceiling.
- Shadow vocabulary: elevation expressed as Tailwind shadow utilities,
  not custom drop shadows. Flag custom box-shadow.
- Motion: transitions use duration-* utilities, not arbitrary ms values.
  Easing via ease-out, ease-in, or ease-in-out. Flag ease-linear.
- Dark-mode coverage: every color utility paired with a dark: variant.
  Flag any leak where dark mode reveals light-only colors.
- Class-strategy dark mode: darkMode: 'class' in config, toggle wired
  through <html class="dark">. Flag prefers-color-scheme reliance.
- Neutral discipline: one neutral scale used consistently. Flag mixing
  (slate + zinc, gray + stone).
- State coverage: hover:, focus-visible:, active:, disabled: all defined
  on every interactive element.

List violations as: Block / Fix-First / Note. Each violation cites the
offending utility and the corrected utility.
```
