# Tailwind-flavored tokens

<Note>
This is a starter variant. Tweak before using on a real project — the values reflect the source design system's defaults, not your specific needs.
</Note>

## Overview

Modeled directly on Tailwind CSS's default theme. The aesthetic is generic-but-opinionated — well-documented defaults that compose cleanly with the wider Tailwind ecosystem. Use this variant when you want maximum flexibility, when the team already runs Tailwind, or when the project's visual identity has not yet been chosen.

## Motion + easing detail

Tailwind ships eight duration steps and four easings, all available as utility classes (`duration-150`, `ease-out`, etc.).

```css
.transition-color { transition: color var(--motion-150) var(--ease-out); }
.transition-bg    { transition: background-color var(--motion-200) var(--ease-out); }
.transition-card  { transition: box-shadow var(--motion-200) var(--ease-out), transform var(--motion-200) var(--ease-out); }
.transition-modal { transition: transform var(--motion-300) var(--ease-in-out), opacity var(--motion-300) var(--ease-out); }
```

Defaults: `--ease-out` for entrances and hover. `--ease-in-out` for symmetric transitions. Avoid `--ease-linear` — it is a rookie tell.

## Why these values

- **Tailwind interop is the whole point.** Mapping these tokens to `tailwind.config.js` gives you utility classes for free. No translation layer.
- **Defaults are documented.** Every value above is in the Tailwind docs. No guessing, no undocumented magic numbers — newcomers have a public reference.
- **Class-strategy dark mode keeps full control.** `<html class="dark">` toggles the variant. Better than `prefers-color-scheme` because users override OS preference.
- **Maximum flexibility, opinion-light.** This variant ships no strong stylistic posture. That is the trade-off — pick this when the project's visual identity is not yet chosen.
- **When NOT to use it.** If the product needs a strong opinion (premium-warm like Stripe, sharp-technical like Vercel), pick the matching variant. Tailwind defaults give competent neutral output, not character.
- **Neutral scale is a real choice.** Slate (cool), gray (true-neutral), zinc (neutral with a hint of warm), neutral (chromaless), stone (warm). Pick one and stay with it across the project.

## Claude Design prompts

### Setup prompt

```
You are setting up design tokens for a new project using the Tailwind-flavored variant.

Implement the tokens from /frontend/design-tokens/tailwind in two places:

1. As CSS custom properties on :root in a globals.css file (for non-utility consumers).
2. As theme extensions in tailwind.config.js so utilities like bg-slate-900, text-blue-600, p-6 resolve to the same values.

Example tailwind.config.js theme.extend:

  colors: {
    slate: { 50: 'hsl(210,40%,98%)', 100: 'hsl(210,40%,96%)', 200: 'hsl(214,32%,91%)', /* ... */ },
    blue:  { 500: 'hsl(217,91%,60%)', 600: 'hsl(221,83%,53%)', 700: 'hsl(224,76%,48%)' },
  },
  fontFamily: {
    sans: ['Inter', 'ui-sans-serif', 'system-ui', 'sans-serif'],
    mono: ['JetBrains Mono', 'ui-monospace', 'monospace'],
  },

Dark mode: class strategy. Set darkMode: 'class' in tailwind.config.js. Toggle <html class="dark"> from your theme switcher.

Pick one neutral scale and stay with it: slate (cool), gray (default), zinc (slight warmth), neutral (chromaless), or stone (warm). Do not mix.

Confirm both globals.css and tailwind.config.js are wired before generating any component.
```

### Component prompt

```
Generate a [COMPONENT_NAME] component using the Tailwind-flavored token set.

Aesthetic: opinionated Tailwind defaults. Light by default, dark via the class strategy.

Constraint: use Tailwind utility classes only. No inline styles. No custom CSS classes outside Tailwind utilities. No arbitrary values like p-[13px] - every value comes from the scale.

Produce all states: default, hover (hover:), focus-visible (focus-visible:), active (active:), disabled (disabled:), loading if applicable. Dark variants on every relevant utility (dark:bg-slate-900, dark:text-slate-100, etc.).

Constraints:
- Spacing from the 4px-base scale (p-1 through p-24).
- Radius from {rounded-sm, rounded, rounded-md, rounded-lg, rounded-xl, rounded-2xl}.
- Motion via {transition, duration-{75|100|150|200|300}, ease-out|ease-in-out}.
- One neutral scale (chosen at setup) plus blue/red/green/amber for accents and status.
- Type sizes from {text-xs, text-sm, text-base, text-lg, text-xl, text-2xl, ...}. No arbitrary sizes.

Return the component as plain HTML or a React component. Markup only - no extracted CSS.
```

### Polish review prompt

```
Review this component against the Tailwind-flavored polish bar. Walk every dimension and flag anything below standard.

1. Utility-class consistency - every spacing, color, radius, and motion value comes from a Tailwind utility, not an arbitrary value.
2. Spacing scale - all gaps and padding are from the 4px-base scale (p-1, p-2, ..., p-24). No p-[13px] one-offs.
3. Radius defaults - rounded values from {rounded-sm, rounded, rounded-md, rounded-lg, rounded-xl, rounded-2xl}.
4. Motion - transitions use duration-* utilities, not arbitrary ms values. Easing via ease-out, ease-in, or ease-in-out (no linear).
5. Dark-mode coverage - every color utility has a paired dark: variant. No leaks where dark mode reveals light-only colors.
6. Class-strategy dark mode - is darkMode set to 'class' in config, and is the toggle wired through <html class="dark">?
7. Neutral discipline - is one neutral scale used (slate OR gray OR zinc OR neutral OR stone), not mixed?
8. State coverage - hover:, focus-visible:, active:, disabled: all defined on every interactive element?

For each violation return the offending utility and the corrected utility.
```

## Sample component

## Override examples

Switch the neutral scale from slate (cool) to zinc (slightly warm):

```css
/* before */ --color-slate-900: hsl(222,47%,11%);
/* after  */ --color-zinc-900:  hsl(240,6%,10%);
```

Switch dark mode from class to media-query:

```js
// before — tailwind.config.js
darkMode: 'class',
// after
darkMode: 'media',
```

Tighten the default border radius for a sharper aesthetic:

```js
// tailwind.config.js theme.extend.borderRadius
borderRadius: { DEFAULT: '4px', md: '4px', lg: '6px', xl: '8px' },
```
