# Workflows

A workflow is a full recipe that spans both [Stage 1 and Stage 2](/backend/workflow/mental-model). It tells you what to do from "I have an idea" to "the module is shipped." Each workflow names the steps, the order, and the prompt or doc that drives each step.

The pages below cover the four flows that come up most often.

  </Card>
</CardGroup>

Use these as recipes, not laws. Adapt to context.
