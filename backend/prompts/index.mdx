# Prompts

Prompt pages document the slash commands you run during the backend workflow. Each page has the same four-section structure: when to use, the prompt itself (copy-paste-ready), what you get back, and what to do next.

The prompts split by stage. RIPER prompts run during [Stage 1](/backend/workflow/mental-model) inside Obsidian against Opus. SpecKit prompts run during Stage 2 inside the project's codebase against Sonnet. You never run a Stage 2 prompt before Stage 1 finishes — see [Mental Model](/backend/workflow/mental-model) for why.

## Where these fit in context

These prompts are tools, not goals. They live inside two larger flows:

- **The two-stage workflow** — see [Mental Model](/backend/workflow/mental-model) for why the prompts split into Stage 1 and Stage 2 groups, and why you never cross the boundary mid-module.
- **The new-module workflow** — see [New module](/backend/workflows/new-module) for the end-to-end recipe that strings these prompts together.

Use the cards above to jump to a specific prompt. Use the workflow links to understand when each prompt fires in the sequence.
