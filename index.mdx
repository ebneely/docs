# Welcome

This is the operating manual for how I run Claude (Opus + Sonnet) when building software. It documents the workflow, the prompts, and the decisions — not any specific project.

Every project I build uses this playbook. The playbook itself stays project-agnostic — it's the manual; projects apply the manual.

## What this playbook covers

Two distinct mindsets, two tabs:

<CardGroup cols={2}>
  <Card title="Backend" icon="server">

</Note>

## The wave system

This playbook assumes you know which wave the project is in. Waves bound expectations: at W1 (MVP) the security review is a checklist; at W3 (Scale) it's a full STRIDE threat model. Same agent, different depth, driven by current wave.

| Wave | Stage | MAU range | What changes |
|------|-------|-----------|--------------|
| **W1** | MVP | 0 – 10K | Single Railway instance, basic security checklist |
| W2 | Early Growth | 10K – 100K | Read replicas, formal threat model |
| W3 | Scale | 100K – 1M | Partitioning, pen testing, ASVS L2 |
| W4 | Hyper-scale | 1M+ | Multi-region, continuous threat modeling |

See [**Waves**](/backend/workflow/waves) in the Backend section for the full table and bump procedure.
